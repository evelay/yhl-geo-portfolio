# 报告模块契约

默认报告要系统、详细、完整。正文至少覆盖以下模块，附录输出可执行题库和监测库。

## 正文模块

1. `执行摘要与关键发现`：说明对象、平台、问题规模、P0 意图、内容机会和主要风险。
2. `研究依据与方法升级`：写清参考方法、为什么适用于 AI 搜索意图挖掘，以及本报告的限制。
3. `测试场景与事实校准`：列品牌/产品/价格/合规/竞品事实，标注来源级别和待确认项。
4. `真实数据接入状态与校准模式`：说明哪些数据已接入、哪些只是计划、当前评分是预测还是校准。
5. `输入归一化与对象边界`：归一化种子词、品牌、产品线、人群、区域、竞品、材料来源和排除范围。
6. `用户角色与场景矩阵`：覆盖决策者、使用者、运营者、采购者、合规/IT 等角色。
7. `双层意图地图`：先任务层，再 GEO 操作层，说明核心问题方向和资产用途。
8. `问题簇与优先级评分`：展示聚类结果、问题数量、评分维度、P0/P1/P2 排序，并标注评分是否已用真实数据校准。
9. `五段式查询重写`：口语问法、独立重写、检索短语、证据查询、标题输入。
10. `国内 AI 平台适配`：DeepSeek、豆包、千问、Kimi、元宝的问法风格、采样 Prompt 和追问保留策略。
11. `内容资产与 FAQ 映射`：文章、页面模块、FAQ、知识库、标题输入包和证据补采任务。
12. `监测 Prompt 与采样计划`：平台、频率、变量、记录字段、品牌提及和证据来源记录。
13. `证据缺口与数据校准`：说明需要外接搜索量、客服数据、真实平台采样、销售线索数据或竞品资料校准。
14. `合规与禁止回答边界`：按风险等级列禁止断言、必须提示人工确认的事项。
15. `30/60/90 天落地路线`：把问题库转为内容生产、页面改造、监测和复盘节奏。

## 附录模块

- `问题库`：至少包含 `question_id / cluster / intent_type / question / standalone_rewrite / retrieval_rewrite / evidence_query / content_asset / priority / compliance_level`。
- `评分矩阵`：至少包含十维评分和加权总分。
- `追问链路`：至少包含 `chain_id / root_question_id / parent_question_id / follow_up_level / context_dependency / follow_up_question / standalone_rewrite / platform_fit`。
- `监测 Prompt 库`：至少覆盖 DeepSeek、豆包、千问、Kimi、元宝。
- `内容选题库`：至少包含选题、主问题、目标资产、证据需求、优先级。
- `FAQ 题库`：至少包含问题、回答边界、证据需求、对应页面/知识库。
- `知识库条目建议`：适合沉淀为解释型、流程型、风险型或价格型知识库。
- `真实数据源状态`：记录数据源、接入状态、记录数、时间范围、校准用途和下一步。
- `AI 平台采样计划或真实采样结果`：未采样时输出计划，已采样时输出答案摘要、品牌提及、引用来源和风险断言。
- `数据校准动作`：说明哪些评分维度需要用真实数据重算。

## 完整度下限

- P0 示例报告：正文不少于 15 个模块，问题库不少于 18 条，追问链路不少于 8 条，监测 Prompt 不少于 5 条。
- 生产报告：按输入规模扩展，问题库建议不少于 30 条；无法达到时必须说明数据不足原因。
- 所有事实、价格、合规和竞品判断都必须标注证据状态，不得把生成推断写成已验证事实。
