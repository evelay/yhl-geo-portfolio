# 真实数据接入与校准

本 skill 可以处理真实数据，但不能在未授权、未提供或未连接数据源时声称已经拿到真实数据。

## 数据模式

- `M0 未接入`：只有种子词和业务假设。输出意图空间、采样计划和证据缺口，不能写真实搜索量、真实 AI 答案或真实转化。
- `M1 用户提供`：用户上传客服问答、销售话术、客户访谈、站内搜索、投放词、社群评论、AI 回答截图或表格。可做频次、聚类、证据强度和问题优先级校准。
- `M2 工具/连接器导入`：通过可用工具或用户授权接口读取搜索量、站内搜索、CRM、客服系统、飞书/表格、网站日志或 AI 平台采样结果。必须记录工具、时间、范围和权限边界。
- `M3 已采样校准`：已有国内 AI 平台真实回答、品牌提及、引用来源、搜索量或转化信号。可把评分矩阵从“预测分”升级为“校准分”。

## 可接入数据

- AI 平台采样：DeepSeek、豆包、千问、Kimi、元宝的 Prompt、答案摘要、品牌提及、提及位置、引用来源、风险断言和追问漂移。
- 搜索与内容数据：关键词搜索量、Search Console、站内搜索、页面点击、内容排名、FAQ 点击。
- 客户与销售数据：客服问答、销售话术、客户访谈、CRM 线索、成交/流失原因。
- 社群与投放数据：社群评论、广告搜索词、投放词包、评论区问题、竞品提及。
- 官方与证据数据：产品页、价格页、法律条款、案例、帮助中心和公开 API 文档。

## 输入字段

每个数据源建议记录：

- `source_id`
- `source_type`
- `provider`
- `access_status`：`未接入 / 用户提供 / 已连接 / 已采样 / 待授权 / 不适用`
- `record_count`
- `date_range`
- `calibration_use`
- `next_action`

## 输出规则

- 报告正文必须有 `真实数据接入状态与校准模式`。
- 附录必须有 `真实数据源状态`；如果未接入真实数据，要明确写出原因和下一步。
- 有真实 AI 平台回答时，新增 `AI 平台真实采样结果`；没有时，输出 `AI 平台采样计划`，不得伪造答案。
- 评分矩阵必须说明当前是 `预测评分` 还是 `真实数据校准评分`。
- 证据状态要进入每个 P0 问题的解释，不得让读者误以为示例报告等于真实市场监测。
