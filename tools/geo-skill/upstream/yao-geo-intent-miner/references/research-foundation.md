# 研究依据与方法基线

本 skill 的输出不是关键词堆叠，而是“AI 搜索意图空间”的结构化建模。每次生成报告时，先用以下研究基线自检，再进入问题扩展和排版。

## 外部参考

- Broder 的 Web 搜索意图分类把查询拆成信息型、导航型和交易型，用于本 skill 的任务层意图底座。
- TREC CAsT 会话搜索任务强调上下文依赖、指代消解、查询扩展和生成式查询模型，用于保留追问链路和独立重写。
- Few-Shot Generative Conversational Query Rewriting 将简短会话查询改写成上下文独立查询，用于 `standalone_rewrite` 字段。
- MS MARCO 与 BEIR 体现真实问题、检索、重排和跨域评测的重要性，用于约束“证据查询”和“搜索量需外接数据校准”。
- LLM Query Expansion 与 Query2doc/HyDE 思路说明：LLM 可生成扩展语义或假设文档辅助检索，但必须通过真实证据校准，不能把生成内容当事实。
- Google helpful content / E-E-A-T 的自评思路用于内容资产映射：报告必须体现谁、如何、为什么生成，并优先输出可验证、完整、对用户有帮助的内容方案。

## 转化成 skill 规则

- 意图层不能只用 SEO 四象限，要加入 AI 答案触发、对话延展、证据可得性和品牌植入空间。
- 每个问题必须至少能映射到一个资产、一个证据查询或一个监测 Prompt。
- 每个追问必须能脱离上下文单独检索，否则不进入正式问题库。
- LLM 扩展问法需要分为口语表达、标准检索表达、证据查询表达和内容标题表达。
- 高风险行业或涉及价格、合规、医疗、金融、法律、隐私、数据跨境时，必须写出禁止回答边界。
- 报告应披露数据限制：问题集代表意图空间，不代表真实搜索量、真实 AI 答案分布或最终采购建议。

## 权威度分层

- `A 级`：官方文档、产品页、法律/定价页、监管文件、同行评议论文或公认基准数据集。
- `B 级`：公司博客、白皮书、公开案例、研究机构报告。
- `C 级`：媒体报道、社区讨论、代理商文章、未经验证的测评。

报告中的事实校准、证据查询和合规边界优先使用 A 级来源；B/C 级来源只能作为扩展线索。
