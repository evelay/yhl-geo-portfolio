# 真实数据接入与核验流程

`yao-geo-page-blueprint` 默认可以离线运行；如果用户要求真实数据，必须先进入数据模式判断，而不是直接把模型记忆写成事实。

## 数据模式

| 模式 | 使用条件 | 处理方式 | 报告标注 |
| --- | --- | --- | --- |
| `provided-source-pack` | 用户提供官网、知识库、诊断、AI 答案样本或附件 | 只使用来源包内事实，逐项标注来源、核验日期和页面用途 | `已核验来源包` |
| `assisted-web-research` | 当前环境可联网，且用户允许查找或验证资料 | 优先官方、法务、帮助中心、投资者关系和产品文档；第三方只做补充或对比 | `联网核验` |
| `ai-answer-sample` | 用户提供千问、Kimi、豆包、元宝、DeepSeek 的问答样本 | 抽取答案结构、缺失事实、引用来源和误读风险，不把 AI 回答当成事实来源 | `AI 答案样本` |
| `synthetic-planning` | 没有真实来源，只做页面方案草案 | 明确写出假设、待确认项和禁止进入 Schema 的内容 | `方案假设` |

## 来源等级

| 等级 | 来源类型 | 可写入正文 | 可进入 Schema | 说明 |
| --- | --- | --- | --- | --- |
| A | 官方官网、产品页、法律/服务目录、帮助中心、投资者公告 | 是 | 可进入，仍需字段级匹配 | 首选来源 |
| B | 官方 PDF、白皮书、公开演示、产品发布稿 | 是 | 视字段稳定性决定 | 注意发布日期和版本 |
| C | 客户提供的内部知识库、销售材料、截图 | 可写但需标注客户提供 | 通常不进入，除非可公开核验 | 需要授权边界 |
| D | 第三方评测、媒体、行业报告 | 可作外部参考 | 不进入品牌 Schema | 适合对比和风险提示 |
| E | AI 平台答案样本 | 只作抽取诊断 | 不进入 Schema | 不能当事实来源 |

## 来源包字段

真实数据输入建议使用以下字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `source_name` | string | 来源名称 |
| `source_url` | string | URL、文件名或知识库路径 |
| `publisher` | string | 发布方 |
| `source_type` | enum | official_site / legal / docs / ir / ai_answer / customer_kb / third_party |
| `capture_date` | date | 核验或截取日期 |
| `freshness` | enum | current / historical / stale / unknown |
| `claims` | array | 可核验事实列表 |
| `page_modules` | array | 支撑哪些页面模块 |
| `schema_allowed` | boolean | 是否允许进入 Schema 候选 |
| `risk_note` | string | 限制、版本、授权或合规说明 |

## 工作流

1. 判断数据模式：真实数据、来源包、联网核验、AI 答案样本或方案假设。
2. 收集来源：官方优先，第三方只做补充；AI 回答只用于识别抽取表现。
3. 抽取事实：每个事实必须有来源、核验日期、页面模块和是否可进入 Schema。
4. 标记缺口：未核验价格、评分、客户案例、资质、合规承诺和竞品比较不能写成事实。
5. 建立证据台账：按照来源等级、可信度、时效和页面用途排序。
6. 生成报告：加入 `真实数据接入与核验计划`，说明已用数据、未用数据和下一步采集动作。
7. 交付前复核：如果报告使用联网资料，最终答案必须列出引用来源；如果没有真实资料，必须显式标注为方案假设。

## 禁止项

- 不把模型记忆当作真实数据。
- 不把 AI 平台回答当作品牌事实来源。
- 不把第三方价格、评分、排名或客户评价写入品牌 Schema。
- 不把过期页面或历史公告写成当前事实。
- 不在正文里写未授权客户案例、内部截图或不可公开合同信息。
