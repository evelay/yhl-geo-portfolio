# 输出风险画像

| 风险 | 表现 | 控制方式 |
| --- | --- | --- |
| 方法过度经验化 | 只按营销直觉排版，没有研究依据 | 必须加入研究依据映射表 |
| 分析不完整 | 只有模块名称，没有输入假设、query fan-out、实体关系和验收标准 | 质量门要求覆盖十个分析维度 |
| 虚假真实数据 | 用模型记忆、AI 答案或第三方转述当成品牌事实 | 引入数据模式、来源等级、核验日期和 Schema 准入规则 |
| Schema 越界 | 写正文没有的评分、价格、评价 | Schema 从正文和 CMS 字段生成 |
| 来源弱 | 关键结论没有来源或核验日期 | 建立证据区与来源台账 |
| CTA 干扰 | 强转化模块压过答案和证据 | 弱 CTA 前置，强 CTA 后置 |
| HTML 长报告难导航 | 页面下拉后目录消失，审阅者需要反复回到顶部 | HTML 报告使用 sticky 目录菜单、锚点和移动端横向菜单 |
| 报告视觉粗糙 | 纯表格堆叠、冷灰底、行距松散、层级不清 | 采用 kami 的暖灰边框、油墨蓝强调、serif 标题和紧凑行距 |
| Word 右侧溢出 | 表格、URL、英文产品名或 Schema 字段超出正文宽度 | 固定表格布局、显式列宽、单元格边距、`wordWrap` 和按需软换行 |
| 示例报告滞后 | 规则更新后示例四件套没有同步 | 每次规则变更后重新生成四格式 |
