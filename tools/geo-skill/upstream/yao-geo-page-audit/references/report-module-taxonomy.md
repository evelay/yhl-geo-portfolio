# 系统化报告模块分类

页面 GEO 诊断报告必须完整回答“诊断了什么、依据是什么、风险在哪里、怎么改、如何验收”。除非用户明确要求轻量版，默认使用下列模块。

## 必备模块

1. 执行摘要：结论、P0/P1 风险、预期收益和关键边界。
2. 输入与范围：目标 URL、品牌名、关键词、目标问题、竞品页、日志/CMS/源码可用性。
3. 页面组合：首页、代表性一级页、代表性二级页或替代页面的选择依据。
4. 公开答案素材问题集：按用户高意图问题列出页面应提供的官方素材，不默认分析具体 AI 平台召回。
5. 权威证据台账：页面观察、官方材料、标准依据、研究依据和无法判断的输入缺口。
6. 抓取与渲染：状态码、robots、sitemap、canonical、meta robots、初始 HTML、渲染 DOM、JS 依赖。
7. 移动端与性能：移动优先内容一致性、首屏稳定、核心内容是否懒加载、LCP/INP/CLS 复测建议。
8. 结构规范性：标题层级、`main`/`article`、目录、摘要、FAQ、表格、列表、面包屑、内链、锚文本。
9. 内容证据：实体全称、数据、案例、引用、作者、来源、更新时间、价格、服务边界、地域/合规约束。
10. AI 可抽取性：原子事实、键值对、对比表、步骤、Q&A、段落独立性、上下文无关摘要、chunk 引用准备度。
11. Schema 一致性：schema 类型、字段来源、正文回溯、缺失字段、冗余字段和校验工具。
12. 来源权威与可信度：账号/作者责任、官方出处、编辑原则、sameAs、百科/媒体误引风险。
13. 代码层修复清单：字段、HTML、schema、meta、robots、sitemap、SSR/SSG、验收命令。
14. 内容结构改造：页面模块、FAQ、事实表、案例、来源标注、中文适配、竞品替代表达边界。
15. 优先级与路线图：P0/P1/P2、负责人、成本、依赖、验收方式和发布批次。
16. 自检与质量报告：四件套同源、排版、链接、证据完整性、未验证风险。

## 模块输出要求

- 每个核心模块至少给出观察、影响和行动，不输出空泛建议。
- P0/P1 清单必须能交给开发或内容负责人执行。
- 对国内答案场景的建议必须说明对应问题意图和可引用素材位置；没有平台采样时不写平台表现。
- 报告必须保留无法判断项，不能用猜测填补日志、CMS、AI 平台召回、排名、答案频次或引用份额数据。
- Word 表格最多 5 列；超过 5 个字段时拆成多张表或字段卡片。

## 完整性自检

| 自检项 | 通过条件 |
|---|---|
| 范围完整 | 至少说明首页、一级页、二级页或替代选择 |
| 证据完整 | 重要结论均有页面、官方、标准、研究或输入缺口标记 |
| 技术完整 | 覆盖抓取、渲染、移动、结构、schema、性能风险 |
| 内容完整 | 覆盖实体、事实、来源、时间、价格、边界、FAQ |
| AI 完整 | 覆盖抽取、chunk、问答、引用准备度和公开答案素材 |
| 交付完整 | 四件套存在且通过 layout 脚本 |
